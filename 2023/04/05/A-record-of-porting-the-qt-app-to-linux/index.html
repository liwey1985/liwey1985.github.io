    一个 Qt 程序移植到 Linux 平台的记录

    工作速记 Qt Linux适配
    发布时间 : 2023-04-05 16:18
    阅读 :
    评论:
    1. linux 下网络函数缺少 htonll 和 ntohll 这两个方法，需要自己补上实现。实现如下：
    #ifdef Q_OS_LINUX
#include <netinet/in.h>
#include <stdint.h>

inline uint64_t htonll(uint64_t hostlonglong)
{
    uint64_t returnVal = hostlonglong;
    if (1 != htonl(1)) {
        char * src = (char*)&hostlonglong;
        char * dst = (char*)&returnVal;
        dst[0] = src[7];
        dst[1] = src[6];
        dst[2] = src[5];
        dst[3] = src[4];
        dst[4] = src[3];
        dst[5] = src[2];
        dst[6] = src[1];
        dst[7] = src[0];
    }

    return returnVal;
}

inline uint64_t ntohll(uint64_t netlonglong)
{
    uint64_t returnVal = netlonglong;
    if (1 != ntohl(1)) {
        char * src = (char*)&netlonglong;
        char * dst = (char*)&returnVal;
        dst[0] = src[7];
        dst[1] = src[6];
        dst[2] = src[5];
        dst[3] = src[4];
        dst[4] = src[3];
        dst[5] = src[2];
        dst[6] = src[1];
        dst[7] = src[0];
    }

    return returnVal;
}
#endif

    1. 套接字函数 int setsockopt(int sockfd, int level, int optname,const void *optval, socklen_t optlen) 第3个参数的常量值名Linux平台与其它两个平台有的不一样。SO_NOSIGPIPE 在 Linux 下应为 MSG_NOSIGNAL

    2. 链接阶段报：Qt cannot find -lGL
      原因是 Qt 找不到 OpenGL 的动态链接库（libGL.so），可以使用 find /usr -name libGL* 找到相对应的动态库路径，然后使用 ln -s /somepath/libGL.so.1 /usr/lib/libGL.so 创建链接到 /usr/lib 下面

    3. Qt编译报错误“GL/gl.h:No such file or directory”
      这是由于系统中没有安装OpenGL库导致，这里我直接把 mesa 相关的开发包都装上了，使用了命令 sudo apt isntall lib*-mesa-dev。PS: 感觉这样应该也同时解决了问题 3，但未验证。

    4. Unknown module(s) in QT: x11extras
      这个是 qxtglobalshortcur 引入的，这在qt6已经没有该模块了
